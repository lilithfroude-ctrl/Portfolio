Hands-on Exercise 2: Car Price Prediction

Part 1: Problem Definition & EDA

Background Information

1. What is the problem?

The primary problem is to build a regression model that accurately predicts the Manufacturer's Suggested Retail Price (MSRP) of a vehicle. The model will use a dataset of over 10,000 automobiles and their various attributes (e.g., Make, Model, Year, Engine HP, Vehicle Size) as explanatory variables to predict the continuous target variable, MSRP.

2. Who might be interested in this problem and why?

Several groups would be interested in this model:

  • Consumers: A potential car buyer could use this model to determine if a listed price for a new or used car is fair, helping them negotiate a better deal.
  • Dealerships: Car dealerships could use the model to set competitive and profitable prices for their inventory.
  • Manufacturers: Automakers can leverage this model to understand what features drive price, informing their decisions on new model design, feature packages, and pricing strategies.
  • Insurance Companies: Insurers could use the predicted MSRP as a factor in determining a vehicle's value for a policy, helping to set premiums and assess total loss payouts.

3. What does the data look like?

To answer this, we will load the car_pricing.csv file and perform an initial inspection to understand its columns, data types, and identify any immediate issues like missing data.

Part 2: Simple Regression & Feature Selection

Three Simple Regression Models

We will now train three simple linear regression models, each using only one explanatory variable. A simple linear regression model takes the form: $Y = \beta_0 + \beta_1X$

  • $Y$ is the target variable (MSRP)
  • $X$ is the single explanatory variable
  • $\beta_0$ is the intercept (the value of Y when X is 0)
  • $\beta_1$ is the parameter estimate (the slope), which tells us how much Y changes for a one-unit increase in X.

To choose our first two features, we will look at the correlation matrix we generated earlier and pick the two variables with the highest absolute correlation with MSRP.

For the third model, we will create a linear-log model, as requested. This model takes the form $Y = \beta_0 + \beta_1\log(X)$. It's used when the relationship between X and Y is non-linear, and it's particularly good when X is a value (like Engine HP) that has diminishing returnsan increase from 100 to 200 HP has a bigger price impact than an increase from 500 to 600 HP.

# --- Helper Function to Train and Evaluate Models ---
def train_and_evaluate(X_train_feat, y_train_data, X_test_feat, y_test_data, model_name):
    print(f"\n--- Results for Model: {model_name} ---")

    # 1. sklearn model for metrics
    lr = LinearRegression()
    lr.fit(X_train_feat, y_train_data)
    y_pred = lr.predict(X_test_feat)

    mae = mean_absolute_error(y_test_data, y_pred)
    mse = mean_squared_error(y_test_data, y_pred)
    r2 = r2_score(y_test_data, y_pred)

    print(f"Metrics (on Test Set):")
    print(f"  MAE (Mean Absolute Error): ${mae:,.2f}")
    print(f"  MSE (Mean Squared Error): ${mse:,.2f}")
    print(f"  R-squared: {r2:.4f}")

    # 2. statsmodels for parameter estimate
    X_train_sm = sm.add_constant(X_train_feat)
    ols_model = sm.OLS(y_train_data, X_train_sm).fit()

    print(f"\nParameter Estimate (from OLS on Training Set):")
    feature_name = X_train_feat.columns[0]
    coefficient = ols_model.params[feature_name]
    p_value = ols_model.pvalues[feature_name]

    print(f"  Feature: {feature_name}")
    print(f"  Coefficient (Slope): {coefficient:.4f}")
    print(f"  P-value: {p_value:.4g}")

    if p_value < 0.05:
        print(f"  Interpretation: For a one-unit increase in '{feature_name}', the MSRP is predicted to increase by ${coefficient:,.2f}.")
    else:
        print(f"  Interpretation: The relationship is not statistically significant.")

    return ols_model

Forward/Backward Variable Selection

Our simple models performed poorly, with high errors and low R-squared values. This is expected, as a car's price is complex and depends on more than one feature.

We will now use sequential feature selection to build a better multiple regression model. This method iteratively adds (forward) or removes (backward) features and builds a model, selecting the combination that performs best.

A Critical Modeling Choice: Our EDA showed that MSRP is heavily right-skewed, but log_MSRP is normally distributed. Linear regression models assume a normally distributed target variable. Therefore, we will build our models to predict log_MSRP, as this will almost certainly result in a more accurate and stable model.

When we evaluate, we will:

  1. Predict the log_MSRP.
  2. Convert this prediction back to the original dollar scale using np.expm1() (the inverse of log1p).
  3. Calculate MAE and MSE using these converted dollar-scale predictions. This gives us a real-world error amount.
  4. Report the R-squared value from the log-scale model, as it's a more accurate measure of the variance explained by the model in its own terms.
In[]:
# We will build models to predict the log-transformed target
y_train_to_use = y_train_log
y_test_to_use = y_test_log

# Instantiate the base linear regression model
lr = LinearRegression()

print("--- Starting Feature Selection (running on a single core) ---")
start_time = time.time()

# --- 1. Forward Selection ---
sfs_forward = SequentialFeatureSelector(
    lr,
    n_features_to_select=8,
    direction='forward',
    cv=5  # 5-fold cross-validation
)
sfs_forward.fit(X_train, y_train_to_use)
forward_selected_features = list(X_train.columns[sfs_forward.get_support()])
print(f"\n--- Forward Selection Results (8 features) ---")
print(f"Selected Features: {forward_selected_features}")

# Evaluate this model
X_train_forward = sfs_forward.transform(X_train)
X_test_forward = sfs_forward.transform(X_test)
lr_forward = LinearRegression()
lr_forward.fit(X_train_forward, y_train_to_use)
y_pred_log_forward = lr_forward.predict(X_test_forward)
y_pred_forward = np.expm1(y_pred_log_forward) # Convert back to dollars

mae_forward = mean_absolute_error(y_test, y_pred_forward)
mse_forward = mean_squared_error(y_test, y_pred_forward)
r2_forward = r2_score(y_test_to_use, y_pred_log_forward) # R2 on log scale

print(f"MAE: ${mae_forward:,.2f}")
print(f"MSE: ${mse_forward:,.2f}")
print(f"R-squared (on log-scale): {r2_forward:.4f}")

# --- 2. Backward Selection ---
sfs_backward = SequentialFeatureSelector(
    lr,
    n_features_to_select=8,
    direction='backward',
    cv=5
)
sfs_backward.fit(X_train, y_train_to_use)
backward_selected_features = list(X_train.columns[sfs_backward.get_support()])
print(f"\n--- Backward Selection Results (8 features) ---")

# Evaluate this model
X_train_backward = sfs_backward.transform(X_train)
X_test_backward = sfs_backward.transform(X_test)
lr_backward = LinearRegression()
lr_backward.fit(X_train_backward, y_train_to_use)

r2_backward = r2_score(y_test_to_use, y_pred_log_backward) # R2 on log scale

print(f"MAE: ${mae_backward:,.2f}")
print(f"MSE: ${mse_backward:,.2f}")
print(f"R-squared (on log-scale): {r2_backward:.4f}")

Part 3: Final Regression Model & Conclusion

Final Model

Based on our analysis, the clear choice for the final model is the one identified by both forward and backward feature selection.

Variables Chosen: The final set of 8 predictors is:

  • Year
  • Engine HP
  • Engine Cylinders
  • Popularity
  • Vehicle Size Encoded
  • Driven_Wheels_all wheel drive
  • Driven_Wheels_front wheel drive
  • Driven_Wheels_rear wheel drive

This model is a log-linear model (it predicts the log of MSRP). This choice is strongly justified by our EDA, which showed log(MSRP) was normally distributed, and by the feature selection results, which produced a model with a vastly higher R-squared (0.77) than models built on the original MSRP (0.37).

We will now formally train this final model and report its metrics.

Conclusion

1. Are you satisfied with your final models performance? Why or why not?

I am satisfied with the improvement, but not the final absolute performance.

  • Why satisfied: The final model's R-squared of 0.7735 is a massive leap from our best simple model (0.3679). We are explaining 77% of the variance in log(MSRP) vs. 37% of MSRP. The MAE also dropped from over $22.4k to $15.8k. This shows that using a log-transformed target and multiple features was the correct approach.
  • Why not satisfied: An average error of $15,830 is still very high in a practical sense. For a $40,000 car, being off by that much makes the model unsuitable for precise pricing. It indicates that while we've captured the main trends, we are still missing key information.

2. What do you propose for the next steps?

To improve this model, I would propose three key next steps:

  1. More Advanced Feature Engineering: We dropped several columns that contain rich information. My next step would be to engineer features from Make (grouping by luxury, standard, etc.), Model, and especially the Market Category column, which we dropped due to missing data. We could parse its comma-separated values into new binary features (e.g., is_Luxury, is_High-Performance).
  2. Try Non-linear Models: Linear regression assumes a linear relationship, which may not hold true. I would test more complex, non-linear models like a Random Forest Regressor or Gradient Boosting (XGBoost). These models can capture complex interactions between features and often require less feature preprocessing.
  3. Check for Multicollinearity: I would run a VIF (Variance Inflation Factor) check on our final predictors. Engine HP and Engine Cylinders are likely highly correlated, which can make the model's coefficients unstable and hard to interpret.

3. What would you do differently if you could?

If I were to start over, I would not have dropped the Make, Model, and Market Category columns at the beginning. I would have spent more time in the EDA phase cleaning and encoding them, as they almost certainly contain significant predictive power that our current model is missing. The Make (e.g., BMW vs. Toyota) is a huge driver of price.

4. Based on your findings what would you want to communicate to business leadership in this industry?

I would communicate three main points:

  1. Price is Logarithmic: A car's price doesn't increase in flat dollar amounts; it increases by percentages. Our model proves that predicting log(MSRP) is far more effective. This means adding a feature (like a sunroof) adds a percentage of value, not a fixed $1,000.
  2. Engineering & Age are Key: The most powerful predictors are Engine HP, Engine Cylinders, and Year. This confirms that performance and age are the dominant, most easily quantifiable drivers of price.
  3. Model is Good for Trends, Not Pricing: The current model, with its ~$16k average error, is a good baseline for understanding broad market trends (e.g., "how much does 100 HP add to a car's value, generally?"). It is not accurate enough to be used for setting specific prices on individual cars. To achieve that, we would need to invest in the "Next Steps" mentioned above, specifically by incorporating Make, Model, and other categorical data.

            > jupyter nbconvert notebook1.ipynb notebook2.ipynb
