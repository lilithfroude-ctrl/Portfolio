1. Problem Introduction & Motivation

This project aims to build a machine learning model to predict the presence of heart disease in patients based on a set of medical attributes. The problem is a binary classification task, where the model will learn to distinguish between two classes: patients with heart disease (target = 1) and patients without heart disease (target = 0).

The motivation for this model is significant. Heart disease is a leading cause of death globally, and the ability to accurately predict its presence using readily available patient data can be a powerful tool for early diagnosis. An effective model could assist clinicians in identifying high-risk patients who may require further testing or immediate intervention, potentially improving patient outcomes and reducing healthcare costs.

The dataset provided contains 303 observations and 6 columns. The target variable is heart_disease, and the five predictor variables are age, sex, max_heart_rate, angina_level, and non_anginal_pain. Our goal is to use these predictors to build a trustworthy model that accurately classifies the target.

2.3. Handling Missing Data

From the .info() output, we identified missing values in the age and angina_level columns (and possibly max_heart_rate after cleaning). We will handle these as follows:

1. age (Numerical): We will impute the missing values using the median age. The median is more robust to outliers than the mean.

2. max_heart_rate (Numerical): We will also impute any missing values here with the median.

3. angina_level (Categorical): We will impute the missing values using the mode (the most frequent value).

In[3]:

3. Model Selection

(No code needed for this section)

For this binary classification problem, a Logistic Regression model is a strong and appropriate initial choice.

Reasoning:

Interpretability: Logistic Regression is a linear model that is highly interpretable. It provides coefficients for each feature, allowing us to understand how each variable (like age or sex_male) contributes to the predicted probability of heart disease.

Efficiency: It is computationally simple and fast to train, making it an excellent baseline model.

Probabilistic Output: It directly outputs a probability (from 0 to 1) of a patient having heart disease, which is often more useful in a medical context than a simple "yes" or "no" answer. A doctor might treat a patient with a 90% predicted probability very differently than one with a 55% probability.

While other models like Decision Trees, Random Forests, or Support Vector Machines (SVMs) could also be used, Logistic Regression provides the best balance of simplicity, interpretability, and performance for an initial analysis.

4. Model Evaluation

(No code needed for this section)

To evaluate the performance of our classification model, we cannot rely on accuracy alone, especially in a medical context where the cost of a wrong answer can be high. We would use the following methods:

1. Train-Test Split

First, we must split our data into a training set (to build the model) and a testing set (to evaluate it on unseen data). This prevents overfitting and gives a realistic measure of performance.

2. Confusion Matrix

This $2 \times 2$ table is the foundation for most other metrics. It shows:

  • True Positives (TP): Correctly predicted "Heart Disease."
  • True Negatives (TN): Correctly predicted "No Heart Disease."
  • False Positives (FP): Incorrectly predicted "Heart Disease" (a "false alarm").
  • False Negatives (FN): Incorrectly predicted "No Heart Disease" (a "missed diagnosis").

3. Key Evaluation Metrics

  • Accuracy: $(TP + TN) / (Total)$. The percentage of all predictions that were correct. It can be misleading if the classes are imbalanced.
  • Precision: $TP / (TP + FP)$. Of all patients the model predicted had heart disease, how many actually did? High precision minimizes false positives.
  • Recall (Sensitivity): $TP / (TP + FN)$. Of all patients who actually had heart disease, how many did the model correctly identify? In this medical case, recall is likely the most important metric, as failing to identify a sick patient (a False Negative) is often more dangerous than a false alarm (a False Positive).
  • F1-Score: The harmonic mean of Precision and Recall $(2 \cdot (Precision \cdot Recall) / (Precision + Recall))$. It provides a single score that balances both metrics.

4. ROC Curve (Receiver Operating Characteristic) and AUC (Area Under the Curve)

  • The ROC curve plots the True Positive Rate (Recall) against the False Positive Rate at various probability thresholds.
  • The AUC score (from 0 to 1) measures the entire area under this curve. A score of 1.0 is a perfect model, while a score of 0.5 is no better than random guessing. This metric is very robust as it evaluates the model's performance across all possible thresholds.
